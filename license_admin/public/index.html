Designed for PowerDNS with MySQL/MariaDB

PDNS Console is a self‑hosted admin interface for PowerDNS using the gmysql backend. It operates in Native mode only (no MASTER/SLAVE UI), which keeps your setup simple and cluster‑friendly.

  • Point the UI at the same PowerDNS database your nameservers use.
  • Run multiple web nodes behind a load balancer – sessions are database‑backed.
  • Forward and reverse zones with PTR support and zone‑aware filtering.
  • DNSSEC lifecycle management and DS publication assistance.
  • Dynamic DNS endpoint for A/AAAA updates (ddclient compatible).
Native Mode Architecture
Authoritative servers share one database. PDNS Console manages zones/records directly in that DB.
Safe by Default
NS/SOA handled automatically; conflict checks; server‑side validation is authoritative.
No External Dependencies
Offline license validation; no phone‑home. Runs wherever you run PowerDNS.

Frequently Asked Questions

What platforms are supported?

Linux servers running PHP 8+, MySQL/MariaDB, and PowerDNS with the gmysql backend.

Does it require the PowerDNS API?

No. PDNS Console works directly with the PowerDNS database (gmysql) in Native mode.

How is licensing enforced?

A local, offline key validates free vs. commercial limits. Free tier manages up to 5 zones.

Is DNSSEC fully supported?

Yes. You can enable DNSSEC, generate keys, perform safe rollovers, and get DS assistance.

Quick Start
  1. Install PowerDNS with MySQL/MariaDB backend (gmysql) in Native mode.
  2. Import the provided schema and configure database credentials.
  3. Create your super admin user via CLI, then log in to the UI.
